# 灵感栈 App 总结

## 1. 当前产品定位

灵感栈是一款面向个人用户的本地优先 AI 资源库。
它不是传统笔记本，也不是单纯的 Prompt 收藏夹，而是把三类常用 AI 资产统一放到一个地方：

- 提示词 Prompt：适合直接复制使用的现成模板
- 技能 Skill：适合反复复用的方法说明、工作流和能力定义
- MCP：适合把 AI 连接到 GitHub、文档、数据库、浏览器等外部工具的数据接入配置

这套产品的核心价值，是让用户不必在聊天记录、收藏夹、GitHub 项目和零散文档之间来回翻。
用户可以先从官方目录里挑现成资源，再把自己长期会用的版本导入进来，最后统一搜索、收藏、复制和复用。

## 2. 目标用户

当前版本最适合这几类用户：

- 经常使用 ChatGPT、Claude、Gemini、Cursor、Cline、Cherry Studio 的个人用户
- 需要反复写 Prompt、整理工作流或配置工具接入的开发者
- 有内容创作、写作、办公整理、研究学习、求职准备需求的高频 AI 用户
- 想把常用 AI 资源沉淀成自己的“个人工具箱”的用户

## 3. 当前已经做成的核心模块

### 首页

首页已经不是单纯的“后台工作台”，而是一个资源指挥台。
它承担三件事：

- 让用户快速理解这款产品是做什么的
- 让用户直接搜索任务、场景或平台
- 让用户通过精选合集和资源分类快速开始使用

首页当前包含：

- 主视觉与品牌入口
- 资源搜索框
- 官方资源数、合集数、我的导入数、目录同步状态
- 新手入门入口
- Prompt / Skill / MCP 三类资源解释卡
- 精选合集
- 新手推荐、热门资源、分类预览、最近收藏

### Prompt 模块

Prompt 模块已经支持：

- 官方提示词目录浏览
- 分类筛选和标签筛选
- 搜索
- Prompt 详情页
- 变量填写
- 预览与复制
- 收藏
- 本地导入

Prompt 的定位是“马上复制就能用”。

### Skill 模块

Skill 模块当前已经支持：

- 官方技能目录浏览
- 分类筛选和标签筛选
- 搜索
- 技能详情页
- 输入要求、使用步骤、模型支持说明
- 复制技能内容
- 高级信息折叠区
- 收藏
- 本地导入

Skill 的定位是“把做事方法沉淀下来，后面重复用”。

### MCP 模块

MCP 模块当前已经支持：

- 官方 MCP 目录浏览
- 分类筛选和标签筛选
- 搜索
- MCP 详情页
- 支持客户端、接入步骤、环境变量、配置模板、安全提醒
- 复制配置模板
- 收藏
- 本地导入

MCP 的定位是“把 AI 接到外部工具和数据源上”。

### 我的资源

“我的”页承接本地个人资产：

- 我的收藏
- 我的导入
- 本地长期复用资源

这保证了官方目录和个人资产可以共存，但不会互相覆盖。

## 4. 当前资源规模

当前官方目录已经扩到：

- Prompt：110
- Skill：106
- MCP：116
- 精选合集：8

这已经具备“可用资源库”的基础规模，而不是演示级样板。

## 5. 当前技术架构

### 前端

- Flutter
- Riverpod
- go_router
- Drift + SQLite
- flutter_secure_storage
- Dio

当前架构是本地优先。
即使服务端没开，App 也可以先从本地官方目录种子启动和使用；有服务时，再走官方目录动态下发更新。

### 后端

- Go
- SQLite
- 内置官方目录 bootstrap
- 目录下发接口

后端当前的职责不是做复杂 SaaS，而是提供一个轻量的官方目录服务和未来同步能力的基础。

### 目录链路

目前目录链路已经打通：

- Flutter 端本地 seed
- catalog 导出脚本
- contracts/catalog/bootstrap.json
- Go 服务内置 bootstrap.json

也就是说，移动端和后端使用的是同一套官方目录数据结构。

## 6. 当前交互和体验状态

当前版本已经解决了几个关键问题：

- 全中文界面
- 对小白更友好的文案
- 分类和标签筛选
- 兼容中文输入法的搜索提交逻辑
- 本地数据库自恢复
- Windows 调试构建可运行

同时，界面视觉已经建立了比较稳定的 AI-native 风格：

- 冷白灰底色
- 靛蓝强调色
- 毛玻璃卡片
- 统一圆角和柔和阴影
- 更接近原生应用的搜索框和底部 Dock

## 7. 当前品牌与视觉方向

当前命名已经确定为：

- 中文名：灵感栈
- 英文名：LingStack

这个名字的含义很清楚：

- “灵感”对应 Prompt、创意、思路、表达
- “栈”对应技能、工具、连接方式、资产沉淀

也比较适合作为 GitHub 仓库、产品名和后续品牌扩展的统一锚点。

## 8. 当前版本仍然存在的边界

虽然现在已经是一套完整原型，但它还不是最终成熟产品。
当前边界主要有这些：

- 资源数量虽然已过百，但还可以继续往更专业、更分职业、更分平台的方向扩
- 官方目录还没有独立 CMS
- 还没有真正的用户账号体系和跨设备同步
- 还没有做语义搜索、推荐排序和个性化发现
- 还没有完成 Android 模拟器 / 真机自动化回归链路
- 品牌图标和视觉素材刚建立，后面还可以继续统一到应用启动图、商店图和宣传图里

## 9. 下一步最值得做的事

如果按产品价值排序，下一阶段最值得做的是：

1. 继续扩官方资源库，尤其是按职业、平台、目标任务扩展
2. 做更强的合集系统，例如按“开发者 / 内容创作者 / 运营 / 求职者 / 学生”分入口
3. 完善导入体验，让个人资源更容易进入自己的资源库
4. 增加真正的目录更新说明和版本感知
5. 在 Android 设备上跑完整交互回归

## 10. 一句话总结

灵感栈现在已经不是一个“看起来像 AI 工具”的原型，而是一套方向明确、结构完整、资源规模达标、能继续往产品化推进的个人 AI 资源库基础版本。
